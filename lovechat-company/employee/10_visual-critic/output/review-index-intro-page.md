## 問題点（優先度順）

### 🔴 CRITICAL（今すぐ直す）

#### 1. リード文が長すぎて読まれていない

```html
<p class="lead">恋愛のメッセージ/DMには、続く人・失速する人の「会話のクセ」がある。
20の質問で、あなたの会話タイプ・相性・事故りやすいポイントと、相性キャラがわかる。</p>
```

- 2文・74文字は診断前のリードとして長すぎる
- ユーザーはボタンを押すか押さないかを0.5秒で決める
- 「事故りやすいポイント」は具体的すぎてかえって引く

**修正案（30文字以内にする）:**
```
あなたの恋愛メッセージ、どのクセに当てはまる？
```
または
```
20問で、あなたの恋愛メッセージのクセがわかる。
```

---

#### 2. バッジ「LOVECHAT 16 ｜ 恋愛メッセージ診断」がブランドの顔なのに弱い

現在: 小さなbadgeクラス → 控えめ過ぎる
ユーザーが最初に目に入るのはH1だが、サービス名のインパクトが弱い

**修正案:**
- バッジサイズを現在の1.2倍に
- ピンクのグロー効果を薄く追加（`text-shadow: 0 0 12px rgba(255,93,143,.4)`）

---

### 🟡 MAJOR（次の反復で直す）

#### 3. H1の「どのタイプ？」ハイライトがイマイチ

`<span class="hl">どのタイプ？</span>` — ハイライトはあるが視覚的に弱い
占いカード感を出すなら、グラデーションテキストにすべき

**修正案:**
```css
.hl {
  background: linear-gradient(135deg, var(--accent), var(--accent2));
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
  background-clip: text;
}
```

#### 4. スタートボタンの下のnote（免責文）が目立ちすぎる

```html
<p class="note">※この診断は人格を断定するものではなく...</p>
```
- フォントサイズと色を確認
- ボタンとnoteの間に余白が足りない場合、ボタンへの視線誘導が弱まる
- ボタン押下率に直接影響する

**修正案:** 
- noteを `font-size: 10px; opacity: 0.5;` に下げる（読ませる文ではない）

#### 5. チップ（⏱ 約90秒 / 📝 全20問 / 💞 16キャラ判定）の順番を変えるべき

現在: `⏱ 約90秒` → `📝 全20問` → `💞 16キャラ判定`

ユーザー心理の優先順位: 「どんな結果が得られるか」→「手間はどれくらいか」

**修正案:** `💞 16キャラ判定` → `⏱ 約90秒` → `📝 全20問`

---

### 🟢 MINOR（余裕があれば）

#### 6. introCharsのキャラ表示が良い — ただしもう少し大きくても良い
- 現在: `52px × 52px`
- スマホで8キャラ横並びのため小さく見える
- `60px × 60px` + `gap: 4px` に変更すると存在感が出る

#### 7. 「診断をはじめる」ボタンのコピーが弱い

「診断をはじめる」は機能説明。感情を動かすコピーに変えると押下率が上がる。

**修正案（選択肢）:**
- 「あなたのタイプをさっそく見る →」
- 「恋愛メッセージのクセを診断する」
- 「90秒で診断スタート」

## A → S にするための追加提案

1. **キャッチコピーを H1 の上に置く**
   - バッジの代わりに「あなたの恋愛メッセージには、クセがある。」という大きめの煽り文
   - H1 でタイプ診断への誘導
   - 3段構成: キャッチコピー → H1 → CTAボタン

2. **背景にうっすらキャラクターを散らす**
   - introChars をフル表示するのもよいが、背景に淡く16体が散らしてあると占いカード感が増す

3. **診断前のミニティーザー**
   - 「●●型の人は〜」という典型例を1つだけ見せて興味を引く
   - 押下率が大幅に上がる可能性あり
