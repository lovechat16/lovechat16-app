# 診断設計レビュー v2｜全20問の品質分析
作成: 01_diagnosis-planner
日付: 2026-06-08
対象: index.html の BANK（AR/FS/EC/DP 各5問 = 合計20問）

---

## 総評

**B評価（使用可能だが改善余地あり）**

4軸の設計意図は正しく、各軸に5問の設問は適切。
ただし「軸間の混入」と「重複設問」が一部あり、診断精度を下げている可能性がある。

---

## 軸別レビュー

### AR軸（積極 vs 受け身）

| Q# | 設問 | 評価 | コメント |
|----|------|------|---------|
| AR-1 | 「会話が途切れたら？」 | ✅ 良好 | 積極性を測る典型的な設問 |
| AR-2 | 「気になる人ができたら？」 | ✅ 良好 | 積極性の動機に焦点、明確 |
| AR-3 | 「メッセージのラリー中、自分は？」 | ⚠️ 注意 | AR-1と被る。「ラリーを広げる vs 答える」はAR-1「話題を振る vs 待つ」と実質同じ |
| AR-4 | 「デートに誘うときは？」 | ✅ 良好 | リアルアクション（デート誘い）での積極性を測る。他の設問と差別化できている |
| AR-5 | 「しばらく連絡が途切れた相手には？」 | ✅ 良好 | AR-2と設定が違う（知らない人 vs 疎遠になった人）。補完的 |

**問題:** AR-1とAR-3が設問の本質において重複している

**改善案（AR-3を変更）:**
```
現在: 「メッセージのラリー中、自分は？」
変更: 「相手への好意を伝えるなら？」
   A: 自分からはっきりと言葉にする
   B: 行動や雰囲気で察してもらう
```
→ 積極性を「行動 vs 待つ」の新しい角度で測れる

---

### FS軸（即レス vs じっくり）

| Q# | 設問 | 評価 | コメント |
|----|------|------|---------|
| FS-1 | 「相手から返信が来たら？」 | ✅ 良好 | 返信速度の直接計測 |
| FS-2 | 「返信を作るときは？」 | ✅ 良好 | プロセス（考える/勢い）を測る。FS-1と補完的 |
| FS-3 | 「通知を見た瞬間は？」 | ✅ 良好 | 「見た→既読」のタイムラグを測る独自角度 |
| FS-4 | 「1日のメッセージ頻度は？」 | ⚠️ 注意 | FS-1と重複気味。どちらも「返信の速さ/頻度」を測っている |
| FS-5 | 「即レス、どう思う？」 | ✅ 良好 | 態度・価値観を測る。行動と分離されていて良い |

**問題:** FS-1（返信速度）とFS-4（メッセージ頻度）は相関が高く、同じ軸を二重測定している

**改善案（FS-4を変更）:**
```
現在: 「1日のメッセージ頻度は？」
変更: 「長い文と短い文、送るのは？」
   A: その場のノリで、まず短く送る
   B: ちゃんと伝えたいから、まとめてから送る
```
→ FS軸の「即vs熟考」を「長さ/質」の角度から測れる

### EC軸（感情豊か vs 落ち着き）

| Q# | 設問 | 評価 | コメント |
|----|------|------|---------|
| EC-1 | 「文面の絵文字・スタンプは？」 | ✅ 良好 | 感情表現の手段を直接測る |
| EC-2 | 「楽しいときは？」 | ✅ 良好 | 特定感情（楽しい）時の表現を測る |
| EC-3 | 「好意を感じたときは？」 | ✅ 良好 | 感情を「出す/隠す」の社会的コントロールを測る |
| EC-4 | 「相手が落ち込んでいたら？」 | ⚠️ 注意 | これはECというよりEMPATHY（共感スタイル）に近い。しかし実用上EC軸の指標として機能する |
| EC-5 | 「自分のメッセージ、言われがちなのは？」 | ⚠️ 注意 | 自己申告バイアスあり。「クール」の方が「かっこいい」と思って選ぶ人が出る可能性 |

**問題:** EC-5は社会的望ましさバイアスが入りやすい

**改善案（EC-5を変更）:**
```
現在: 「自分のメッセージ、言われがちなのは？」
変更: 「ミスをしてしまった時、LINEで？」
   A: 「ごめん…！」と感情を乗せて謝る
   B: 「ごめん、以後気をつける」とシンプルに
```
→ 感情表現を具体的シナリオで測ることで、望ましさバイアスを排除

### DP軸（深掘り vs 広がり）

| Q# | 設問 | 評価 | コメント |
|----|------|------|---------|
| DP-1 | 「距離を縮めたいときは？」 | ✅ 良好 | 距離の縮め方（深さ vs 軽さ）を正確に測る |
| DP-2 | 「好きな話題は？」 | ⚠️ 注意 | DP-4と重複気味 |
| DP-3 | 「会話に満足するのは？」 | ✅ 良好 | 満足の基準（本音 vs テンポ）を測る。独自角度 |
| DP-4 | 「やり取りの初期は？」 | ⚠️ 注意 | DP-2と被る（どちらも「深いvsライト」を測る） |
| DP-5 | 「重い話題が来たら？」 | ✅ 良好 | 受動的状況でのDP傾向を測れる |

**問題:** DP-2「好きな話題」とDP-4「やり取りの初期は」は実質同じこと（深いvsライトの話を好むか）を聞いている

**改善案（DP-2を変更）:**
```
現在: 「好きな話題は？」
変更: 「会話の中で、自分からする質問は？」
   A: 「それってどうしてそう思うの？」と深掘りする
   B: 「そういえばさ！」と話題を広げる
```
→ DP軸の「深さ vs 広がり」を質問行動で直接測れる

## 総括改善提案

### 確認された問題
1. AR-1 ≈ AR-3（会話主導権の重複測定）
2. FS-1 ≈ FS-4（返信速度・頻度の重複測定）
3. EC-5（自己申告バイアス）
4. DP-2 ≈ DP-4（話題の深さの重複測定）

### 優先対応
- **高優先**: EC-5の変更（バイアス排除 → 診断精度向上）
- **中優先**: DP-2の変更（重複排除）
- **低優先**: AR-3、FS-4の変更（機能しているが改善余地あり）

### 実装注意事項
- 設問変更は BANK オブジェクトの編集（index.html）
- ただし index.html は保護ファイルのため、07_dev-engineer への引き継ぎが必要
- 設問変更は診断結果分布に影響する可能性があるため、変更前後で結果分布を確認すること
