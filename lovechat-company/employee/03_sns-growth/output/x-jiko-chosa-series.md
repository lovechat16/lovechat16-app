# X「恋愛事故調査ファイル」シリーズ
作成日: 2026-06-08
担当: 03_sns-growth
ツール: kanri.html（恋愛事故調査ファイル下書き生成）

## このシリーズについて

kanri.htmlで生成した「恋愛事故調査ファイル」をベースにした投稿シリーズ。
実際の（または架空の）チャット事例を匿名化・タイプ分析してX投稿する。

**バズ理由**：
- 「あるある」で強い共感が生まれる
- 「これ私の話じゃ？」で拡散される
- 「タイプ診断してみたい」への誘導が自然

---

```
【恋愛事故調査ファイル #XX】

相手タイプ: ○○型（XXXX）
事故の種類: ○○による○○

─── チャット再現 ───
○○さん（自分）：[メッセージ]
相手：[メッセージ]
○○さん：[メッセージ]
（既読 → 24時間以上返信なし）

─── 分析 ───
このタイプへの○○は「○○」と映る。
○○すれば防げた事故。

相手のタイプを事前に知ってたら違った。
→ 相手のチャットタイプを診断する [URL]

#恋愛事故調査 #ラブチャット診断
```

### 事例01「既読温存型への追いメッセージ事故」

```
【恋愛事故調査ファイル #01】

相手タイプ: 既読温存型（RSCP）
事故の種類: 追いメッセージによる距離の急拡大

─── チャット再現 ───
自分：「昨日の話、どう思った？」
　　　（既読 → 2時間後）
自分：「怒ってる？」
　　　（既読 → 45分後）
自分：「返してよ」
　　　（既読 → 3時間後）
相手：「ごめん、ちょっと忙しくて」
　　　（その後、返信がゆっくりになった）

─── 分析 ───
既読温存型は「今じゃない」という感覚を持っている。
プレッシャーをかけるほど、返しにくくなる。

正解は「1通送ったら返ってくるまで待つ」。
急用なら「急がなくていいけど確認したかった」の一言だけ。

相手のタイプを知ってたら防げた事故。

→ 相手の返信タイプを診断する [URL]

#恋愛事故調査 #ラブチャット診断 #既読スルー
```

## 活用方法

### 投稿リズム
- 週1〜2本を目安に定期連載
- カレンダーの「あるある」投稿と交互に使う
- フォロワーが増えたら「あなたの事故事例を募集」でUGCに展開

### kanri.htmlとの組み合わせ
1. 実際のユーザーからDMで相談を受ける（LINE公式アカウントが窓口）
2. kanri.htmlに貼り付けて匿名化・タイプ推定を実行
3. 生成された下書きをこのフォーマットに整えてX投稿

### 注意事項
- 特定個人が特定されないように必ず匿名化する
- kanri.htmlの「公開前チェックリスト」を必ず確認
- 実在の事例を使う場合は本人の同意が望ましい
- フィクション/合成事例として作成しても問題ない（「よくある事例を元に再構成」など）
