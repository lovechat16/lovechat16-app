## ROUND 1：前提の徹底検証

### 仮説「比較リスト型が最初の投稿に最適」を疑う

buzz-to-post-strategy.md では投稿案04（言語化型）を最初に出すべきと結論した。  
本当にそれが正しいか、反証から考える。

**反証1: アカウント0フォロワー問題**  
アカウント作成直後はフォロワーゼロ。  
言語化型「"返事は考えてから送りたい"派、ほぼ全員が既読スルーに見られてる。」は、  
リーチする人がいなければそもそもインプレッションが発生しない。  
→ **最初の投稿の目的は「バズること」ではなく「アカウントを正しく育てるための基盤づくり」かもしれない。**

**反証2: プロフィールが完成する前に投稿しても意味がない**  
投稿を見た人がプロフィールを見に来て「診断リンクがない」「bio が空」だと離脱する。  
最初の投稿前にやることがある：
- bio に「16タイプ恋愛LINE診断 / あなたの返信グセを当てます」
- ヘッダー画像（ブランド感）
- 固定ツイートに診断URL

**反証3: 投稿案04 のリスク再評価**  
「既読スルーに見られてる」というフレーズ—これは当たってる感はあるが、  
一部ユーザーに「責められてる感」を与えるかもしれない。  
→ 0フォロワー時代の最初の投稿でネガティブ感情を呼びすぎると、  
RTより「わかる…つらい」コメントが集まりやすく、拡散よりエンゲージメント型になる。

**自己検証1の結論:**  
→ 最初の投稿は「言語化型」でいいが、責め寄りではなくユーモア寄りに調整すべき。  
→ 投稿の前に「プロフィール完成」が条件。これをCEOに伝える必要がある。

## ROUND 2：アルゴリズム構造の推論

### Xのタイムライン表示ロジックを逆算する

**観測事実（Coworkレポートより）:**
- 万バズ投稿の多くはRT重み付けが高い
- 返信・引用が多いと「会話中の投稿」として推薦される

**推論の連鎖:**  
1. 0フォロワー状態では「おすすめ」タブに乗るのが最初の課題  
2. おすすめに乗るには初速（最初の1〜6時間のエンゲージメント率）が重要  
3. エンゲージメント率 = (いいね+RT+返信+引用) ÷ インプレッション  
4. 0フォロワーのインプレッションはほぼゼロ  
5. **したがって最初の数投稿は「フォロワーゼロでも種を蒔く」戦略が必要**

**種の蒔き方3パターン:**

パターンA: ハッシュタグ活用型  
`#MBTI #恋愛診断 #INFJ` などのニッチタグで検索流入を狙う  
→ 即時効果は小さいが確実な流入源になる  
→ ただしハッシュタグだけに頼るとスパム判定リスク

パターンB: 会話参加型  
人気ツイートの返信欄に自然に参加してプロフィール露出を増やす  
→ 本アカウントでやると「プロモ感」が出てブロックされる  
→ 個人アカウントとの連携が前提になる（CEO判断事項）

パターンC: 相互引用誘発型  
「あなたはどっち？」など参加型にして初期フォロワーが引用RTしやすい設計  
→ これが最も有機的にアルゴリズムに乗る可能性が高い

**自己検証2の結論:**  
→ 最初の数投稿は「参加型」（投稿案10系）を混ぜるべき  
→ 純粋な言語化型だけでは初期インプレッションが取れない可能性  
→ 投稿順の見直し案: `プロフィール完成 → 固定ツイート（診断URL）→ 参加型1本 → 言語化型1本 → 比較型1本`

## ROUND 3：競合ポジショニングの深掘り

### 「恋愛診断系アカウント」の地雷パターンを特定する

既存の恋愛診断X アカウントが避けられる理由を推論する。

**地雷パターン1: 恋愛テク屋**  
「既読スルーされたらこう返せ」型の投稿。  
→ 女性ユーザーから「男の攻略法っぽい」と感じられスルーされる  
→ ラブチャット16はこれに絶対寄せない

**地雷パターン2: 自己啓発MBTI**  
「あなたのタイプは〜です。このタイプの人は〜な才能があります」  
→ 「診断商材」感が出て信頼を失う  
→ MBTIモドキの固定人格系も同様

**地雷パターン3: 恋愛強者マウンティング**  
「モテる人はこうしてる」「こんな返信する人は脈なし」  
→ ユーザーを評価する立場に立つと反発される  
→ ラブチャット16は「評価しない」が強み

**ラブチャット16の差別化ポイント（再確認）:**  
1. 「あなたを評価しない、ただ当てる」スタンス  
2. 言い当て感 > 診断正確性  
3. 毒は「自分のことを言われた感」であり攻撃性ではない  
4. 16タイプという多様性 → 「自分のタイプ」を探す楽しさ

**自己検証3の結論:**  
→ 全投稿に「あなたを責めているわけじゃない、ただ当てるだけ」の空気が必要  
→ bio やプロフィールにも「評価しない、ただ当てる」の一文を入れるべき（CEO提案）

## ROUND 4：投稿カレンダーの非線形思考

### 月曜・水曜・金曜・土曜の配置は本当に最適か

buzz-to-post-strategy.md の週次カレンダー：  
月曜=比較型、水曜=言語化型、金曜=参加型、土曜=謎かけ型

**反証アプローチ: 「行動文脈」で見直す**

月曜朝 → 仕事始め、移動中スクロール → 「わかる感」で共感されやすい  
→ ✅ 言語化型の方が月曜朝に向いている可能性

水曜昼 → 週中だるさピーク → 「診断やってみよう」欲求が上がる？  
→ 診断誘導型の方が水曜に向いている可能性

金曜夜 → 週末モード、友達と話したくなる → 参加型・会話誘発型 → ✅ 現状正しい

土曜昼 → ゆっくりスクロール、謎かけに反応する時間がある → ✅ 現状正しい

**修正案:**  
月曜: 言語化型（「わかる感」で週始めの共感を取る）  
水曜: 診断誘導型 OR 比較型（週中の探し物欲求に対応）  
金曜: 参加型（週末の会話発火）  
土曜: 謎かけ型（ゆっくり考える時間に対応）  
→ buzz-to-post-strategy.md の月曜・水曜の型を入れ替えるべき

## ROUND 5：コンテンツの寿命と再利用の設計

### 1本の投稿をどう「資産」に変えるか

**現状の問題:**  
投稿案10本があるが、それぞれが独立している。  
→ 10本投稿したら終わり、次が作れない状態になるリスク

**解決策の推論:**

タイプライブラリ化  
「RSCP/既読温存型ってどんな人？」という投稿を作るとき、  
16タイプ × 投稿型 = 160本の組み合わせがある。  
1週1タイプ特集にすれば16週間=4ヶ月のコンテンツが生まれる。

RTされたら派生を出す  
言語化型が1000RT超えた場合 → 「RTしてくれた人へ：あなたは何タイプ？」を出す  
→ RTがさらなる投稿のトリガーになる仕組みを最初から設計する

診断へのブリッジ投稿  
「自分がどのタイプか気になった人は→診断URL」を定期的に挿む  
→ 毎回URLを貼るのではなく、週1回程度で自然に誘導

**自己検証5の結論:**  
→ コンテンツ設計を「スポット」から「シリーズ」に変える必要がある  
→ 最初の10本を投稿した後の「次の10本」を見越した設計をすべき  
→ 「16タイプ週1紹介」シリーズを02 Result Copywriterへ発注する
