## 現在の最優先仮説（2026-06-09時点）

### 仮説1：「命名」が最も再現性が高い
根拠：「会議の10分前症候群」「5分前の後悔」など、CEOが実体験から出した命名に即座に共感反応が得られた。
→ **誰もが知ってるけど名前がない現象を命名する投稿が最強**

### 仮説2：X（旧Twitter）はURLなし投稿がリーチ3〜5倍
根拠：アルゴリズム観測。URLがあるとインプレッション抑制される。
→ **診断URLは返信欄に置く。本文には入れない。**

### 仮説3：「コメントしやすい終わり方」が拡散を決める
根拠：参加型選手権（「あなたは？」で終わる）がコメント数を増やし、アルゴリズム評価を上げる。
→ **「あなたは？」「名前つけて」「コメントで教えて」で終わる投稿を意図的に混ぜる**

## 06-08リサーチから追加した型（2026-06-09 更新）

### 新型E：「集約型言い切り」
```
構造：長い感情・時間・状況を最短の言葉に圧縮して終わる。説明しない。
根拠：二宮和也「最後の最後まで嵐。5人で嵐。」(1.7M likes) の構造。
芸能人だから伸びたのではなく、感情の圧縮率が異常に高かったから伸びた。
例：
返信を待っている間じゅう
ずっとあなたのことを考えていた。
送信から既読まで、全部。
```

### 新型F：「ツンデレ落ち型」
```
構造：感情（悲しみ・愛情・恥ずかしさ）の直球描写 → 最後の一行で「ざまぁ」「なんやねん」などの笑い・照れ隠しに転換
根拠：「リゾートホテル予約→ざまぁww」(144K)。感動を笑いに転換するラスト1行。
例：
スマホを1時間に12回確認した。
自分で数えてみた。
ざまぁ。
```

## 構造分析：今後試したい型

### 新型A：「現象の解剖型」
```
構造：「〇〇」という現象がある → 理由を2〜3行で解剖 → 「わかる人いる？」
例：
「メール書き直し病」というものがある。
送信前に読み返すと、最初の一文から直したくなる。
完璧主義というより、送信ボタンを押したくないだけだと最近気づいた。
```

### 新型B：「あの人の内側を代弁する型」
```
構造：「〇〇な人って、本当は〜〜なだけだと思う」
例：
返信が遅い人は、あなたの言葉をちゃんと読んでる。
読んで、考えて、どう答えるか迷ってるうちに時間が経ってる。
雑に返せる相手にはすぐ返せるから。
```
狙い：「わかってほしかった」「代わりに言ってくれた」でコメント・引用爆発

### 新型C：「観察記録型」
```
構造：フィクション風実話。短くて背景がある。オチなし。
例：
カフェで隣の人が
一言も話さずに2時間いた。
帰り際に「行こうか」だけ言って一緒に出ていった。
仲良いのかわからなかったけど、なんか好きだった。
```
狙い：「語られすぎてない」ことが想像を生む。保存・引用されやすい。

### 新型D：「全員に当てはまる一言型」
```
構造：説明ゼロ。一行か二行。読んだ人が勝手に自分のことだと思う。
例：
「言わなかったこと」の方が
「言ったこと」より全然多い。
```
狙い：解釈の余地がある→引用RTで「これ私だわ」が連鎖する

## メモ：CEOから拾った生命名2件

> 「会議の10分前症候群」  
> 会議が始まる10分前が一番急に仕事が進む現象。締め切りの相対性理論。

> 「5分前の後悔」  
> 送信ボタンを押した5分後に「違う言い方があった」と気づく現象。誰でも持っている。治らない。

→ どちらも即TIER1。CEOの体験談から命名させるシリーズは定期的に仕込む価値あり。
→ 「名前のない感情をシェアして。名前つけます」という参加型企画に転化できる。
