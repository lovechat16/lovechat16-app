## 1. ストック全体の問題点（構造的欠陥）

### 問題①：型の均一化
`x-type-posts.md`（16本）は全て同じ構文。

```
【ラブチャット診断】
○○型 の人は…
・あるある1
・あるある2
・あるある3
「これ私」ってなる人はRTして → URL
```

3投稿目以降、フォロワーが「またこれか」で止まる。
RTするのは「これだけ私に刺さった」という驚きが必要。均一フォーマットは驚きを殺す。

### 問題②：URL過多
URLが入っている投稿が多すぎる。
2026年のXアルゴリズムは外部リンクの多い投稿のリーチを下げる。
→ URL入り投稿は週2本以内に抑える。残りは共感・会話誘発型にする。

### 問題③：「フォローする理由」が不明確
現状のストックを見ても「このアカウントを毎日見る理由」が弱い。
「診断サイトの宣伝アカウント」に見える。
→ 「このアカウントをフォローすると、恋愛LINEの読み方が変わる」という価値が必要。

### 問題④：万バズ型の最重要パターンが未実行
分析で「最優先」と評価した型が、ストックに落ちていない。

| 万バズ型 | 重要度 | ストックへの落とし込み |
|---|---|---|
| 対比リスト型（144K〜270K） | ★★★ | ほぼなし |
| 造語命名型（10.2Mインプ） | ★★★ | ゼロ |
| 逆張り断定一行型（67K〜229K） | ★★★ | 弱い（あるある01が近いが力不足） |
| 参加型（170K〜254K） | ★★★ | 投稿案03のみ（これは良い） |

### 問題⑤：冒頭1行が弱い
全ストックの冒頭を確認した。スクロールを止める力がない。

悪い例（現状）：
- 「LINEの返信でわかる、あなたの恋愛タイプ🔥」→ 広告
- 「「既読ついてるのに返信来ない」」→ 普通
- 「ねえ見てる？って先に送ってくる人の心理」→ まあまあ

良い冒頭とは：読んだ瞬間「自分の話だ」か「誰かのことだ」と感じる1行。


## 5. 「バズ投稿設計の原則」まとめ

### 必ず守ること

1. **冒頭1行で「自分の話だ」と思わせる**
2. **URL入りは週2本まで**
3. **1投稿1テーマ。詰め込まない**
4. **締めに問いかけを入れる**（コメント誘発）
5. **感情の着地を「安心」か「笑い」か「驚き」にする**（不安煽りはしない）

### 避けること

1. 「✅」「👇」「続出中」→ 広告臭
2. 「女性はこう、男性はこう」→ 分断を生む
3. 「○○のコツ5選」→ テク屋になる
4. ハッシュタグ3個以上→ スパム判定リスク
5. 毎回同じ構文→ 飽きられる
