作成: 08_design-director + 10_visual-critic 協働
日付: 2026-06-08

---

## 概要

SNSシェア時に表示される OGP画像（1200×630px）の設計要件。
Xシェア・LINEシェア・note・Instagramでのシェア画像として機能する。

---

## 共通デザイン仕様

### サイズ・フォーマット
- **サイズ**: 1200 × 630px（Twitter card summary_large_image 最適）
- **フォーマット**: JPG（80%品質）または WebP
- **ファイル名**: `assets/ogp.jpg`

### カラーパレット
```
背景グラデーション: #1a1130 → #2b1645（左上→右下 160deg）
アクセント1: #ff5d8f
アクセント2: #a06bff
テキスト: #f4eeff
ミュート: #b6a8d6
```

### フォント
- メインコピー: 太字 システムフォント（Hiragino / Yu Gothic / Noto Sans JP）
- サブコピー: 細字 システムフォント

## 実装方法（07_dev-engineerへ）

### 現状
- `assets/ogp.jpg` が存在しない（シェア時に画像なし）
- GitHub Pages デプロイ後に og:image を絶対URLに変更が必要

### ショートカット実装案（デプロイ前）
HTMLキャンバスを使ってブラウザ内でOGP画像を動的生成し、
`<meta>` タグを動的に書き換える方法。ただしOGPはサーバーが事前取得するため効果なし。

→ **結論: 静的画像ファイルを `assets/ogp.jpg` に配置することが必須**

### 作業手順
1. デザイン担当がCanvaまたはFigmaで1200×630pxの画像を作成
2. `assets/ogp.jpg` として保存
3. GitHub Pages デプロイ
4. 全HTMLの `og:image` を絶対URL（`https://[user].github.io/[repo]/assets/ogp.jpg`）に変更
5. 07_dev-engineerが一括置換で対応
